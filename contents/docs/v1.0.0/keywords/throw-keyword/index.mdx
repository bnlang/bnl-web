# <I18nEnglish>throw Keyword</I18nEnglish>
# <I18nBangla>নিক্ষেপ কীওয়ার্ড</I18nBangla>


<I18nEnglish>
Use **`throw`** to signal an exceptional condition. It stops normal execution and jumps to the nearest matching `catch`.  
Prefer throwing **`Error`** objects (or subclasses) so stack traces and metadata are preserved.
</I18nEnglish>

<I18nBangla>
**`নিক্ষেপ`** ব্যবহার করুন ব্যতিক্রমী অবস্থায় ত্রুটি জানাতে। এটি স্বাভাবিক এক্সিকিউশন থামিয়ে নিকটতম `ধরুন` ব্লকে চলে যায়।  
স্ট্যাক‑ট্রেস ও তথ্য ধরে রাখতে **`Error`** অবজেক্ট (বা তার সাবক্লাস) নিক্ষেপ করা উত্তম।
</I18nBangla>

## Syntax

<I18nEnglish>

```bnl
throw expression;                 // usually an Error
throw new Error("Message");       // recommended
class ValidationError extends Error {}
throw new ValidationError("Bad input");
```
</I18nEnglish>

<I18nBangla>

```bnl
নিক্ষেপ অভিব্যক্তি;                 // সাধারণত Error
নিক্ষেপ new Error("বার্তা");         // সুপারিশকৃত
class ValidationError extends Error {}
নিক্ষেপ new ValidationError("ভুল ইনপুট");
```
</I18nBangla>

// 1) Basic throw/catch
try {
  const age = -2;
  if (age < 0) throw new Error("Invalid age");
  console.log("OK");
} catch (err) {
  console.error("Error:", err.message);
} finally {
  console.info("Always runs finally");
}

// 2) Custom Error subclass
class InputError extends Error {
  constructor(msg) {
    super(msg);
    this.name = "InputError";
  }
}
function evaluate(x) {
  if (typeof x !== "number") throw new InputError("Number required");
  return x * 2;
}

// 3) Wrapping with cause
function parse(json) {
  try {
    return JSON.parse(json);
  } catch (root) {
    throw new Error("JSON parse failed", { cause: root });
  }
}

try {
  parse("{bad json}");
} catch (err) {
  console.error(err.message);
  if (err.cause) console.info("cause:", err.cause.message);
}

// 4) Throwing in async function rejects the promise
async function download(url) {
  const res = await fetch(url);
  if (!res.ok) throw new Error("Bad response");
  return await res.text();
}

## Notes

<I18nEnglish>
- You can throw **any** value, but prefer `Error` or subclasses for consistent stacks and features like `cause`.  
- `throw` aborts the current execution path and unwinds the stack until a matching `catch`.  
- `finally` runs even if a `throw` happens.  
- Avoid using exceptions for normal control flow; reserve them for truly exceptional situations.  
- When converting errors, **wrap** and set `{ cause: original }` to keep context, or **rethrow** the original (`throw err`) to preserve its stack.
</I18nEnglish>

<I18nBangla>
- যেকোনো মান নিক্ষেপ করা সম্ভব, তবে স্ট্যাক ও কনটেক্সট বজায় রাখতে `Error`/সাবক্লাস ব্যবহার করুন।  
- `নিক্ষেপ` দিলে স্ট্যাক আনউইন্ড হয়ে মিলযুক্ত `ধরুন` পাওয়া পর্যন্ত থেমে যায়।  
- `অবশেষে` ব্লক নিক্ষেপ হলেও চলে।  
- সাধারণ নিয়ন্ত্রণ প্রবাহে ব্যতিক্রম ব্যবহার করবেন না—শুধু বিশেষ পরিস্থিতিতে রাখুন।  
- ত্রুটি রূপান্তর করলে `{ cause: মূল }` দিন, বা মূল ত্রুটিই **পুনরায় নিক্ষেপ** করুন (`নিক্ষেপ ত্রু`) যাতে স্ট্যাক থাকে।
</I18nBangla>
