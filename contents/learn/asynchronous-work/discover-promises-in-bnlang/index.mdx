# <I18nBanglish>Discover Promises in Bnlang</I18nBanglish>

<I18nEnglish>
A **Promise** in **Bnlang** represents a value that may be available now, later, or never.  
It is a structured way to handle asynchronous operations without falling into callback nesting.  
A promise has three states: **pending**, **fulfilled**, and **rejected**.
</I18nEnglish>

<I18nBangla>
**Bnlang** এ **প্রতিশ্রুতি** হলো এমন একটি অবজেক্ট যা কোনো মান এখন, পরে, অথবা কখনো প্রদান করতে পারে।  
এটি কলব্যাকের জটিলতা এড়িয়ে অ্যাসিঙ্ক্রোনাস কাজকে সহজভাবে হ্যান্ডল করতে সাহায্য করে।  
একটি প্রতিশ্রুতিের তিনটি অবস্থা আছে: **pending**, **fulfilled**, এবং **rejected**।
</I18nBangla>

<I18nBanglish>
**Bnlang** e **Promise** holo ekta object jeta ekhon, pore, ba kokhono value dite pare.  
Eta callback er jotojhamela chara async kaj handle korte dey.  
Promise er tin ta state ase: **pending**, **fulfilled**, ar **rejected**.
</I18nBanglish>

## Promise.all

<I18nEnglish>
`Promise.all` runs multiple promises in parallel and resolves when all are fulfilled.  
If any fail, the whole promise rejects.
</I18nEnglish>

<I18nBangla>
`Promise.all` একাধিক প্রতিশ্রুতি একসাথে চালায় এবং সবগুলো পূর্ণ হলে রিজলভ হয়।  
একটি ব্যর্থ হলে পুরো প্রতিশ্রুতি রিজেক্ট হয়।
</I18nBangla>

<I18nBanglish>
`Promise.all` diye ekadik promise ekshathe run hoy, sob fulfill hole resolve hoy.  
Kono ekta fail hole shob reject hoy.
</I18nBanglish>

## Best Practices

<I18nEnglish>
- Always handle errors with `.catch()` to avoid unhandled rejections.  
- Use `Promise.all` or `Promise.race` for running tasks together.  
- Keep promises short and focused for readability.
</I18nEnglish>

<I18nBangla>
- আনহ্যান্ডলড রিজেকশন এড়াতে `.catch()` দিয়ে এরর হ্যান্ডল করুন।  
- একাধিক কাজ চালাতে `Promise.all` বা `Promise.race` ব্যবহার করুন।  
- পড়ার সহজতার জন্য প্রতিশ্রুতি ছোট ও নির্দিষ্ট রাখুন।
</I18nBangla>

<I18nBanglish>
- Unhandled rejection avoid korte `.catch()` use koro.  
- Ekadik kaj run korte `Promise.all` ba `Promise.race` use koro.  
- Readability er jonno promise chhoto ar specific rakho.
</I18nBanglish>
