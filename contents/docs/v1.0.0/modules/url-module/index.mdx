## API Surface

<I18nEnglish>

### WHATWG URL
- **`new URL(input[, base])`** — create URL object.  
- Properties: `href`, `protocol`, `username`, `password`, `host`, `hostname`, `port`, `pathname`, `search`, `hash`.  
- **`url.searchParams`**: interface to manage query params.  
  - `.get(name)`, `.set(name, value)`, `.append(name, value)`, `.delete(name)`.

### Legacy API
- **`url.parse(urlStr)`** — parse into object.  
- **`url.format(obj)`** — format object into string.  
- **`url.resolve(from, to)`** — resolve relative URL.  

</I18nEnglish>

<I18nBangla>
### WHATWG URL
- **`new URL()`** অবজেক্ট তৈরি। প্রপার্টি: href, protocol, host, pathname, search, hash ইত্যাদি।  
- **`searchParams`** দিয়ে কুয়েরি প্যারাম ম্যানেজ করা যায়।

### লিগ্যাসি
- **`parse/format/resolve`** — পুরোনো স্টাইল ফাংশন।
</I18nBangla>

const { URL } = require("url");

// Modern URL API
const myUrl = new URL("https://user:pass@example.com:8080/p/a/t/h?query=1#hash");

console.log(myUrl.hostname);   // example.com
console.log(myUrl.pathname);   // /p/a/t/h
console.log(myUrl.searchParams.get("query")); // 1

myUrl.searchParams.set("query", "42");
console.log(myUrl.toString());

// Legacy API
const url = require("url");
const parsed = url.parse("https://example.com/foo/bar?x=1");
console.log(parsed.hostname, parsed.pathname);

## Notes

<I18nEnglish>
- Prefer **WHATWG URL** for consistency with browsers.  
- `searchParams` provides convenient query manipulation.  
- Legacy API still works but is discouraged for new code.  
- Always provide a **base URL** when parsing relative URLs.  
</I18nEnglish>

<I18nBangla>
- ব্রাউজার সঙ্গতিপূর্ণতার জন্য WHATWG URL ব্যবহার করুন।  
- `searchParams` দিয়ে সহজে কুয়েরি ম্যানেজ করা যায়।  
- পুরোনো API নতুন কোডে ব্যবহার না করাই ভালো।  
- রিলেটিভ URL পার্স করার সময় বেস দিন।  
</I18nBangla>

<I18nBanglish>
- Browser consistency er jonno WHATWG URL use korun.  
- `searchParams` diye easy query manage kora jay.  
- Legacy API new code e use na korai valo.  
- Relative URL parse korte always base provide korun.
