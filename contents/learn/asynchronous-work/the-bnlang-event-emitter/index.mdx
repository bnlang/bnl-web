# <I18nEnglish>The Bnlang Event Emitter</I18nEnglish>
# <I18nBangla>Bnlang ইভেন্ট এমিটার</I18nBangla>
# <I18nBanglish>The Bnlang Event Emitter</I18nBanglish>

<I18nEnglish>
The **Event Emitter** in **Bnlang** is a powerful pattern for handling asynchronous events.  
Objects can **emit** named events, and other parts of the program can **listen** for those events with handlers.  
This decouples event producers from consumers, making code modular and flexible.
</I18nEnglish>

<I18nBangla>
**Bnlang** এ **ইভেন্ট এমিটার** হলো অ্যাসিঙ্ক্রোনাস ইভেন্ট হ্যান্ডল করার একটি শক্তিশালী প্যাটার্ন।  
অবজেক্ট কোনো ইভেন্ট **এমিট** করতে পারে এবং প্রোগ্রামের অন্য অংশ তা **শুনে** হ্যান্ডলার চালাতে পারে।  
এতে প্রডিউসার ও কনজিউমার আলাদা থাকে, ফলে কোড হয় মডুলার ও ফ্লেক্সিবল।
</I18nBangla>

<I18nBanglish>
**Bnlang** e **event emitter** holo async event handle korar ekta powerful pattern.  
Object gulo **emit** korte pare event, onno jaygay handler diye **listen** kora jay.  
Ete producer ar consumer alada thake, code modular ar flexible hoy.
</I18nBanglish>

## Multiple Listeners

<I18nEnglish>
An event can have multiple listeners. All of them will be called in the order they were registered.
</I18nEnglish>

<I18nBangla>
একটি ইভেন্টে একাধিক লিসেনার থাকতে পারে। তারা নিবন্ধিত ক্রমে ডাকা হবে।
</I18nBangla>

<I18nBanglish>
Ekta event e onek gulo listener thakte pare. Tara register korar order e call hobe.
</I18nBanglish>

## Removing Listeners

<I18nEnglish>
You can remove a listener using `.off` or `.removeListener`.  
This is useful when a listener is no longer needed.
</I18nEnglish>

<I18nBangla>
`.off` বা `.removeListener` ব্যবহার করে লিসেনার সরানো যায়।  
যখন কোনো লিসেনারের আর দরকার নেই তখন এটি কাজে আসে।
</I18nBangla>

<I18nBanglish>
`.off` ba `.removeListener` diye listener remove kora jay.  
Listener proyojon na thakle eta use kora hoy.
</I18nBanglish>

## Once Listeners

<I18nEnglish>
Use `.once` to register a listener that runs only the first time the event is emitted.
</I18nEnglish>

<I18nBangla>
`.once` ব্যবহার করলে লিসেনার শুধু একবার চালানো হবে।
</I18nBangla>

<I18nBanglish>
`.once` diye ekbar matro listener run hobe.
</I18nBanglish>

## Best Practices

<I18nEnglish>
- Remove listeners you no longer need to prevent memory leaks.  
- Use `.once` for events expected to occur only once.  
- Keep listener functions small and focused.  
- Use descriptive event names for clarity.
</I18nEnglish>

<I18nBangla>
- মেমোরি লিক এড়াতে অপ্রয়োজনীয় লিসেনার সরিয়ে ফেলুন।  
- যেসব ইভেন্ট একবার ঘটবে সেগুলোর জন্য `.once` ব্যবহার করুন।  
- লিসেনার ফাংশন ছোট এবং নির্দিষ্ট রাখুন।  
- বর্ণনামূলক ইভেন্ট নাম ব্যবহার করুন।
</I18nBangla>

<I18nBanglish>
- Memory leak avoid korte proyojonhin listener remove korun.  
- Ekbar ghotbe emon event er jonno `.once` use korun.  
- Listener chhoto ar focused rakhun.  
- Spashto event name use korun.
</I18nBanglish>
