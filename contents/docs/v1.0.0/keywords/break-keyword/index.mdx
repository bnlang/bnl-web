# <I18nEnglish>break Keyword</I18nEnglish>
# <I18nBangla>থামুন কীওয়ার্ড</I18nBangla>


<I18nEnglish>
The **`break`** statement immediately exits the **nearest** enclosing loop (`for`, `while`, `do`) or a `switch`.  
Control continues with the first statement after that construct.
</I18nEnglish>

<I18nBangla>
**`থামুন`** স্টেটমেন্ট নিকটতম **লুপ** (`for`, `while`, `do`) বা **বিকল্প** (`switch`) ব্লক থেকে সাথে সাথে বের হয়ে আসে।  
এরপরের লাইন থেকে কোড চলতে থাকে।
</I18nBangla>



---

## Syntax

<I18nEnglish>

```bnl
break;
```
</I18nEnglish>

<I18nBangla>

```bnl
থামুন;
```
</I18nBangla>

```bnl
// Stop a for loop early
for (let i = 1; i <= 10; i++) {
  if (i === 5) {
    console.warn("Stopping at 5");
    break;
  }
  console.log("i =", i);
}

// Find first match and break
const nums = [3, 9, 12, 7];
for (const n of nums) {
  if (n % 3 === 0 && n > 10) {
    console.log("Found:", n);
    break;
  }
}

// Break in a while loop
let count = 0;
while (true) {
  if (++count === 3) break;
  console.log("looping", count);
}

// Break in switch
const grade = "B";
switch (grade) {
  case "A":
    console.log("Excellent");
    break;
  case "B":
    console.log("Good");
    break;
  default:
    console.log("Unknown");
}
```

## Notes

<I18nEnglish>
- `break` affects only the **innermost** loop or switch.  
- Use clear conditions to avoid accidental infinite loops that rely on `break`.  
- Prefer early `return` inside functions when you need to exit multiple levels of logic.
</I18nEnglish>

<I18nBangla>
- **`থামুন`** কেবলমাত্র **সবচেয়ে ভেতরের** লুপ/বিকল্পকে প্রভাবিত করে।  
- `থামুন`-র ওপর নির্ভরশীল অসাবধানী অনন্ত লুপ এড়াতে শর্ত পরিষ্কার রাখুন।  
- বহু স্তরের লজিক থেকে বের হতে হলে ফাংশনের ভেতরে আগে **`ফেরত/return`** দেওয়া উত্তম।
</I18nBangla>
