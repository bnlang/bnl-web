# <I18nEnglish>The Bnlang Event Loop</I18nEnglish>
# <I18nBangla>Bnlang ইভেন্ট লুপ</I18nBangla>

<I18nEnglish>
The **event loop** in **Bnlang** is the core mechanism that makes asynchronous programming possible.  
It allows tasks like I/O, timers, and events to run without blocking the main thread.  
The event loop continuously checks for pending tasks and executes them in phases, ensuring programs remain responsive.
</I18nEnglish>

<I18nBangla>
**Bnlang** এ **ইভেন্ট লুপ** হলো এমন একটি মূল ব্যবস্থা যা অ্যাসিঙ্ক্রোনাস প্রোগ্রামিং সম্ভব করে।  
এটি I/O, টাইমার এবং ইভেন্ট চালাতে পারে মূল থ্রেড ব্লক না করে।  
ইভেন্ট লুপ সারাক্ষণ কিউ চেক করে এবং ধাপে ধাপে টাস্ক চালায়, ফলে প্রোগ্রাম রেসপন্সিভ থাকে।
</I18nBangla>

## How the Event Loop Works

<I18nEnglish>
1. Tasks are placed into queues (macrotasks, microtasks).  
2. The event loop checks the call stack. If empty, it processes the next task.  
3. Timers, I/O callbacks, and event handlers are executed.  
4. Microtasks (like promise callbacks) are always executed before the next macrotask.  
5. This cycle continues indefinitely.
</I18nEnglish>

<I18nBangla>
১. টাস্কগুলো কিউতে রাখা হয় (ম্যাক্রোটাস্ক, মাইক্রোটাস্ক)।  
২. ইভেন্ট লুপ কল স্ট্যাক পরীক্ষা করে। ফাঁকা থাকলে পরবর্তী টাস্ক চালায়।  
৩. টাইমার, I/O কলব্যাক এবং ইভেন্ট হ্যান্ডলার চালানো হয়।  
৪. মাইক্রোটাস্ক (যেমন প্রতিশ্রুতি কলব্যাক) সবসময় পরবর্তী ম্যাক্রোটাস্কের আগে চালানো হয়।  
৫. এই চক্র অনন্তকাল ধরে চলতে থাকে।
</I18nBangla>

## Phases of the Event Loop

<I18nEnglish>
The event loop is divided into phases:  
- **Timers Phase** → Executes callbacks from `setTimeout` and `setInterval`.  
- **I/O Callbacks Phase** → Executes deferred I/O callbacks.  
- **Check Phase** → Executes `setImmediate` callbacks.  
- **Close Callbacks Phase** → Executes close events like socket close.  
Between each phase, microtasks (like promises) are executed before moving on.
</I18nEnglish>

<I18nBangla>
ইভেন্ট লুপ কয়েকটি ধাপে বিভক্ত:  
- **টাইমার ধাপ** → `setTimeout` এবং `setInterval` এর কলব্যাক চালানো হয়।  
- **I/O কলব্যাক ধাপ** → স্থগিত I/O কলব্যাক চালানো হয়।  
- **চেক ধাপ** → `setImmediate` কলব্যাক চালানো হয়।  
- **ক্লোজ কলব্যাক ধাপ** → সকেট ক্লোজের মত ক্লোজ ইভেন্ট চালানো হয়।  
প্রতিটি ধাপের মাঝে মাইক্রোটাস্ক চালানো হয়।
</I18nBangla>

## Best Practices

<I18nEnglish>
- Do not block the event loop with heavy computations.  
- Use promises and async/await for clarity.  
- Break large tasks into smaller microtasks.  
- Keep the event loop free for handling I/O efficiently.
</I18nEnglish>

<I18nBangla>
- ভারী কম্পিউটেশনে ইভেন্ট লুপ ব্লক করবেন না।  
- পরিষ্কার কোডের জন্য প্রতিশ্রুতি এবং async/await ব্যবহার করুন।  
- বড় টাস্ককে ছোট মাইক্রোটাস্কে ভাগ করুন।  
- I/O দক্ষভাবে চালানোর জন্য ইভেন্ট লুপ ফ্রি রাখুন।
</I18nBangla>
