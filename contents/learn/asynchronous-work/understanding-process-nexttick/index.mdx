# <I18nEnglish>Understanding process.nextTick</I18nEnglish>
# <I18nBangla>process.nextTick বোঝা</I18nBangla>

<I18nEnglish>
In **Bnlang**, `process.nextTick` is a special function that schedules a callback to run **immediately after the current operation**, before the event loop continues.  
It gives a way to defer execution without waiting for the next event loop phase.  
Think of it as *microtask scheduling*, ensuring the function runs as soon as possible after the current stack clears.
</I18nEnglish>

<I18nBangla>
**Bnlang** এ `process.nextTick` এমন একটি ফাংশন যা বর্তমান অপারেশন শেষ হওয়ার সাথে সাথেই কলব্যাক চালায়, ইভেন্ট লুপের পরবর্তী ধাপের আগেই।  
এটি কোড চালাতে বিলম্ব না করে defer করার উপায় দেয়।  
এটি *microtask scheduling* এর মত কাজ করে, যাতে বর্তমান স্ট্যাক শেষ হবার সাথে সাথেই ফাংশন চালানো হয়।
</I18nBangla>

## Comparison with setImmediate and setTimeout

<I18nEnglish>
- `process.nextTick` → Runs **before** the event loop continues (highest priority microtask).  
- `setImmediate` → Runs in the **check phase** of the event loop (after I/O).  
- `setTimeout` → Runs after a minimum delay in the **timers phase**.
</I18nEnglish>

<I18nBangla>
- `process.nextTick` → ইভেন্ট লুপ চালু হবার আগেই চালায় (সবচেয়ে বেশি প্রায়োরিটি মাইক্রোটাস্ক)।  
- `setImmediate` → ইভেন্ট লুপের **check phase** এ চলে (I/O এর পরে)।  
- `setTimeout` → ন্যূনতম ডিলে শেষে **timers phase** এ চলে।
</I18nBangla>

## Best Practices

<I18nEnglish>
- Use `process.nextTick` for very short deferrals that must run before I/O.  
- Avoid heavy logic inside `nextTick` to prevent blocking other microtasks.  
- Prefer promises (`Promise.resolve().then(...)`) for clearer async code when possible.
</I18nEnglish>

<I18nBangla>
- I/O এর আগে চালাতে হলে `process.nextTick` ব্যবহার করুন।  
- ভারী লজিক `nextTick` এ দেবেন না, এতে অন্য মাইক্রোটাস্ক ব্লক হবে।  
- সম্ভব হলে পরিষ্কার কোডের জন্য প্রতিশ্রুতি (`Promise.resolve().then(...)`) ব্যবহার করুন।
</I18nBangla>
