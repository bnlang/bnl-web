# <I18nEnglish>Understanding setImmediate</I18nEnglish>
# <I18nBangla>setImmediate বোঝা</I18nBangla>

<I18nEnglish>
In **Bnlang**, `setImmediate` is a function that schedules a callback to run on the **check phase** of the event loop.  
This means it executes **after I/O events** but before timers scheduled with `setTimeout(…, 0)`.  
It is useful for deferring execution to the next cycle of the event loop without unnecessary delays.
</I18nEnglish>

<I18nBangla>
**Bnlang** এ `setImmediate` হলো এমন একটি ফাংশন যা কলব্যাককে ইভেন্ট লুপের **চেক ফেজে** চালায়।  
এটি I/O ইভেন্টের পরে, কিন্তু `setTimeout(…, 0)` এর আগেই চালানো হয়।  
এর মাধ্যমে অযথা ডিলে না দিয়ে পরবর্তী লুপ সাইকেলে কোড defer করা যায়।
</I18nBangla>

## setImmediate vs setTimeout

<I18nEnglish>
- `setImmediate` executes callbacks in the **check phase**, right after I/O.  
- `setTimeout(fn, 0)` executes in the **timers phase**, which comes later.  
In practice, `setImmediate` usually runs faster than `setTimeout(…, 0)`.
</I18nEnglish>

<I18nBangla>
- `setImmediate` **চেক ফেজে** চালায়, I/O এর পরপরই।  
- `setTimeout(fn, 0)` **টাইমার ফেজে** চালায়, যা পরে আসে।  
প্র্যাকটিক্যালি, `setImmediate` সাধারণত `setTimeout(…, 0)` এর চেয়ে দ্রুত চলে।
</I18nBangla>

## Best Practices

<I18nEnglish>
- Use `setImmediate` to break up long-running tasks and keep the event loop responsive.  
- Prefer `setImmediate` over `setTimeout(…, 0)` when you need execution as soon as the current I/O completes.  
- Avoid blocking inside callbacks to maintain performance.
</I18nEnglish>

<I18nBangla>
- দীর্ঘ কাজ ছোট ছোট অংশে ভেঙে ইভেন্ট লুপ রেসপন্সিভ রাখতে `setImmediate` ব্যবহার করুন।  
- বর্তমান I/O শেষ হবার সাথে সাথেই চালাতে হলে `setImmediate` ব্যবহার করুন, `setTimeout(…, 0)` এর পরিবর্তে।  
- কলব্যাকের ভিতরে ব্লকিং কোড দেবেন না, পারফরম্যান্স কমে যাবে।
</I18nBangla>
