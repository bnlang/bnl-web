# <I18nEnglish>Bnlang Package Manager (BPM)</I18nEnglish>
# <I18nBangla>Bnlang প্যাকেজ ম্যানেজার (BPM)</I18nBangla>
# <I18nBanglish>Bnlang Package Manager (BPM)</I18nBanglish>

<I18nEnglish>
The **Bnlang Package Manager (BPM)** is the official package manager for the Bnlang programming language.  
It allows developers to **discover**, **install**, and **manage** libraries and tools, making the ecosystem richer and more productive.  
BPM simplifies sharing code and ensures a consistent workflow across projects.
</I18nEnglish>

<I18nBangla>
**Bnlang প্যাকেজ ম্যানেজার (BPM)** হলো Bnlang প্রোগ্রামিং ভাষার অফিসিয়াল প্যাকেজ ম্যানেজার।  
এটি ব্যবহার করে ডেভেলপাররা সহজেই **লাইব্রেরি খুঁজে**, **ইনস্টল করে** এবং **ম্যানেজ করতে** পারে।  
এর ফলে ইকোসিস্টেম আরও শক্তিশালী হয় এবং কাজ হয় দ্রুত।  
BPM কোড শেয়ারিং সহজ করে এবং সব প্রজেক্টে একসাথে কাজ করার অভিজ্ঞতা দেয়।
</I18nBangla>

<I18nBanglish>
**Bnlang Package Manager (BPM)** holo Bnlang programming language er official package manager.  
Eta diye developer ra easy vabe **library khuje**, **install kore**, ar **manage korte** pare.  
Ete ecosystem aro strong hoy ar productivity baray.

## Example Workflow

<I18nEnglish>

```sh
# Initialize a new project
bpm init

# Install a library
bpm install my-package

# Remove it if no longer needed
bpm uninstall my-package

# Publish your own library
bpm publish
```
</I18nEnglish>

<I18nBangla>

```sh
# নতুন প্রজেক্ট শুরু
bpm init

# একটি লাইব্রেরি ইনস্টল
bpm install lodash

# পরে আপডেট করুন
bpm update lodash

# প্রয়োজন না হলে সরিয়ে দিন
bpm remove lodash

# নিজের লাইব্রেরি প্রকাশ করুন
bpm publish
```
</I18nBangla>

<I18nBanglish>

```sh
# Notun project start
bpm init

# Ekta library install
bpm install lodash

# Pore update korun
bpm update lodash

# Dorkar na hole remove korun
bpm remove lodash

# Nijer library publish korun
bpm publish
```
</I18nBanglish>

## Best Practices

<I18nEnglish>
- Keep your <code>bnl_package.json</code> file under version control.  
- Use semantic versioning (<code>^1.2.3</code>) for predictable updates.  
- Always run <code>bpm audit</code> to check for security issues.  
- Share your packages with meaningful names and documentation.
</I18nEnglish>

<I18nBangla>
- আপনার <code>bnl_package.json</code> ফাইল version control এ রাখুন।  
- পূর্বাভাসযোগ্য আপডেটের জন্য semantic versioning (<code>^1.2.3</code>) ব্যবহার করুন।  
- নিরাপত্তা সমস্যা চেক করার জন্য সবসময় <code>bpm audit</code> চালান।  
- প্যাকেজ অর্থপূর্ণ নাম এবং ডকুমেন্টেশন সহ শেয়ার করুন।
</I18nBangla>

<I18nBanglish>
- Nijer <code>bnl_package.json</code> file ke version control e rakhen।  
- Predictable update er jonno semantic versioning (<code>^1.2.3</code>) use korun.  
- Package ke meaningful name ar documentation shathe share korun.
</I18nBanglish>
