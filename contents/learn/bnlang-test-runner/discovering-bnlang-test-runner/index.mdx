# <I18nEnglish>Discovering the Bnlang Test Runner</I18nEnglish>
# <I18nBangla>Bnlang টেস্ট রানার আবিষ্কার</I18nBangla>

<I18nEnglish>
The **Bnlang Test Runner** is the official tool for writing and running tests in the Bnlang ecosystem.  
It provides a simple API for defining tests, assertions, and hooks, ensuring reliable and maintainable code.  
With the test runner, you can automate quality checks and confidently refactor projects.
</I18nEnglish>

<I18nBangla>
**Bnlang টেস্ট রানার** হলো Bnlang ইকোসিস্টেমে টেস্ট লেখার এবং চালানোর অফিসিয়াল টুল।  
এটি টেস্ট, অ্যাসারশন এবং হুক ডিফাইন করার সহজ API দেয়, যা কোডকে নির্ভরযোগ্য ও মেইনটেনযোগ্য রাখে।  
টেস্ট রানার দিয়ে কোডের মান যাচাই অটোমেট করা যায় এবং নিশ্চিন্তে প্রজেক্ট রিফ্যাক্টর করা যায়।
</I18nBangla>

## Assertions

<I18nEnglish>
Assertions are checks that confirm expected outcomes.  
Common methods include `t.equal`, `t.notEqual`, `t.ok`, and `t.fail`.
</I18nEnglish>

<I18nBangla>
অ্যাসারশন মানে হলো প্রত্যাশিত ফলাফল যাচাই করা।  
সাধারণ মেথডগুলোর মধ্যে আছে: `t.equal`, `t.notEqual`, `t.ok`, এবং `t.fail`।
</I18nBangla>

## Hooks

<I18nEnglish>
The runner supports lifecycle hooks:  
- `before` → Runs once before all tests.  
- `after` → Runs once after all tests.  
- `beforeEach` → Runs before each test.  
- `afterEach` → Runs after each test.
</I18nEnglish>

<I18nBangla>
রানার লাইফসাইকেল হুক সাপোর্ট করে:  
- `before` → সব টেস্টের আগে একবার চলে।  
- `after` → সব টেস্টের পরে একবার চলে।  
- `beforeEach` → প্রতিটি টেস্টের আগে চলে।  
- `afterEach` → প্রতিটি টেস্টের পরে চলে।
</I18nBangla>

## Best Practices

<I18nEnglish>
- Group related tests for readability.  
- Keep test functions small and focused.  
- Use hooks for setup/teardown logic.  
- Automate tests in CI/CD pipelines for reliability.
</I18nEnglish>

<I18nBangla>
- সম্পর্কিত টেস্ট একসাথে লিখুন।  
- টেস্ট ফাংশন ছোট ও নির্দিষ্ট রাখুন।  
- সেটআপ/টিয়ারডাউন লজিকের জন্য হুক ব্যবহার করুন।  
- নির্ভরযোগ্যতার জন্য টেস্ট CI/CD তে অটোমেট করুন।
</I18nBangla>
