### 1. Reporting Issues

### 2. Requesting Features

- Explain why the feature is useful and how it aligns with Bnlangâ€™s goals (Bangla, Banglish, and English programming).

### 3. Improving Documentation
- Documentation lives in the `contents/` folder (MDX files).  
- You can help by improving clarity, translations, or adding new examples.

### 4. Contributing Code
1. Fork the repository and clone your fork.  
2. Create a new branch for your change:  
   ```bash
   git checkout -b feature/my-change
   ```
3. Make your changes and commit with a clear message:  
   ```bash
   git commit -m "feat: add Bangla alias for WeakSet"
   ```
4. Push your branch:  
   ```bash
   git push origin feature/my-change
   ```
5. Open a Pull Request (PR) to the `main` branch.
