# <I18nEnglish>switch & case Keywords</I18nEnglish>
# <I18nBangla>বিকল্প ও অবস্থা কীওয়ার্ড</I18nBangla>
# <I18nBanglish>switch & case Keywords</I18nBanglish>

<I18nEnglish>
Use **`switch`** to select one of many branches based on a single expression.  
Each **`case`** is a label to match; use **`break`** to stop execution, and **`default`** as the fallback when no case matches.
</I18nEnglish>

<I18nBangla>
**`বিকল্প`** ব্যবহার করুন একটি অভিব্যক্তির মান অনুযায়ী একাধিক পথের মধ্যে যেকোনো একটি চালাতে।  
প্রতিটি **`অবস্থা`** একটি লেবেল; **`থামুন`** দিয়ে ব্লক শেষ করুন, আর কোনো অবস্থা না মিললে **`অন্যথায়`** চলবে।
</I18nBangla>

<I18nBanglish>
**`bikolpo`** diye ekta expression er value onujayi onekgulo path er modhye ekta choose hoy.  
Protiti **`obostha`** label; cholte thaka bondho korte **`thamun`**, ebong kono case match na korle **`onnothay`**.
</I18nBanglish>

## Syntax

<I18nEnglish>

```bnl
switch (expression) {
  case value1:
    // statements
    break;            // usually stop here
  case value2:
    // statements
    break;
  default:
    // fallback
}
```
<br/>
```bnl
// Grouping multiple cases
switch (x) {
  case 1:
  case 2:
  case 3:
    // handle 1,2,3
    break;
  default:
    // ...
}
```
<br/>
```bnl
// Placing default earlier (falls through if no break)
switch (code) {
  default:
    console.log("Unknown");
    // no break -> falls through
  case "A":
    console.log("Handled A");
    break;
}
```
</I18nEnglish>

<I18nBangla>

```bnl

  অবস্থা মান১:

    থামুন;
  অবস্থা মান২:
    // স্টেটমেন্ট
    থামুন;
  অন্যথায়:
    // ডিফল্ট/ফলব্যাক
}
```

```bnl
// একাধিক অবস্থা একসাথে গ্রুপ

  অবস্থা 1:
  অবস্থা 2:

    // ...

```


</I18nBangla>

## Notes

<I18nEnglish>
- Put **`break`** after each `case` unless you intentionally want **fall‑through**.  
- Only one **`default`** per switch; it can appear anywhere in the block.  
- Use a `switch (true)` pattern to express range logic succinctly, otherwise prefer `if...else` for complex predicates.  
- For simple value-to-result mapping, an **object map** can be faster and more readable than a long switch.
</I18nEnglish>

<I18nBangla>
- প্রতিটি `অবস্থা`র শেষে সাধারণত **`থামুন`** রাখুন, নাহলে **fall‑through** হবে।  
- একটি `বিকল্প` ব্লকে মাত্র একটি **`অন্যথায়`** থাকতে পারে এবং এটি যেকোনো স্থানে রাখা যায়।  
- রেঞ্জ লজিকের জন্য `বিকল্প (true)` কৌশল কার্যকর; খুব জটিল শর্তে `যদি...নাহলে` আরামদায়ক।  
- সরল মান‑থেকে‑ফলাফল ম্যাপিং হলে দীর্ঘ `বিকল্প`‑এর বদলে **অবজেক্ট ম্যাপ** ব্যবহার করুন।
</I18nBangla>

<I18nBanglish>
- Protiti `obostha` por **`thamun`** din, na hole **fall‑through** hote pare.  
- Protiti `bikolpo` te sudhu ekta **`onnothay`** thake, ebong seta block er je kono jaygai thakte pare.  
- Range logic e `bikolpo (true)` upokari; onek complex condition e `jodi...nahole` sohoj.  
- Shohoj value→result map hole long switch er cheye **object map** porishkar.
</I18nBanglish>
