# <I18nEnglish>if Keyword</I18nEnglish>
# <I18nBangla>যদি কীওয়ার্ড</I18nBangla>
# <I18nBanglish>if Keyword</I18nBanglish>

<I18nEnglish>
The **`if`** keyword performs conditional branching in Bnlang.  
A block runs only when its condition evaluates to **true**.  
Chain with **`else if`** and **`else`** to handle multiple paths.
</I18nEnglish>

<I18nBangla>
**`যদি`** ব্যবহার হয় শর্তভিত্তিক ব্রাঞ্চিংয়ের জন্য।  
শর্ত **সত্য** হলে ব্লকের কোড রান হয়।  
একাধিক শর্তের জন্য **`নাহলে যদি`** এবং শেষ বিকল্প হিসেবে **`নাহলে`** ব্যবহার করুন।
</I18nBangla>

<I18nBanglish>
**`jodi`** diye conditional branching kora hoy.  
Condition **true** hole block er code run hoy.  
Multiple condition er jonno **`nahole jodi`** ebong fallback hisebe **`nahole`** use korun.
</I18nBanglish>

---

## Syntax

<I18nEnglish>

```bnl
if (<condition>) {
  // then-block
} else if (<another condition>) {
  // else-if block
} else {
  // else block
}
```
</I18nEnglish>

<I18nBangla>

```bnl
যদি (<শর্ত>) {
  // সত্য হলে চালাও
} নাহলে যদি (<আরেক শর্ত>) {
  // বিকল্প শর্ত
} নাহলে {
  // বাকি সব ক্ষেত্রে
}
```
</I18nBangla>

<I18nBanglish>

```bnl
jodi (<condition>) {
  // true hole
} nahole jodi (<onno condition>) {
  // alternative
} nahole {
  // otherwise
}
```
</I18nBanglish>

## Truthy & Falsy

<I18nEnglish>
Bnlang follows JavaScript-style truthiness. The following are **falsy**:
`false`, `0`, `-0`, `0n`, `""` (empty string), `null`, `undefined`, and `NaN`.  
Everything else is **truthy**.
</I18nEnglish>

<I18nBangla>
Bnlang এ JavaScript-এর মত truthiness কাজ করে। **Falsy** মানগুলি হলো:
`false`, `0`, `-0`, `0n`, `""` (ফাঁকা স্ট্রিং), `null`, `undefined`, `NaN`।  
এর বাইরে বাকি সব **truthy**।
</I18nBangla>

<I18nBanglish>
Bnlang e JS moto truthiness kaj kore. **Falsy** man gulo hoilo:
`false`, `0`, `-0`, `0n`, `""`, `null`, `undefined`, `NaN`.  
Onno sob **truthy** dhora hoy.
</I18nBanglish>

## Best practices

<I18nEnglish>

- Keep one language style (English/Bangla/Banglish) per file for readability.
- Prefer strict comparison `===`/`!==` for predictable checks.
- Extract complex conditions into named variables or functions.
- Use early returns inside functions to avoid deep nesting.
</I18nEnglish>

<I18nBangla>

- পড়ার সহজতার জন্য প্রতি ফাইলে একটিই ভাষা স্টাইল অনুসরণ করুন।
- নির্ভরযোগ্য তুলনার জন্য `===`/`!==` ব্যবহার করুন।
- জটিল শর্ত আলাদা ভেরিয়েবল বা ফাংশনে বের করে নিন।
- গভীর নেস্টিং এড়াতে ফাংশনে আগেভাগে `return/ফেরত` দিন।
</I18nBangla>

<I18nBanglish>

- Readability er jonno protiti file e ekta style e thakun.
- Strict comparison `===`/`!==` prefer korun.
- Complex condition gula variable/function e ber kore nin.
- Deep nesting komate early `return` use korun.
</I18nBanglish>
