# <I18nEnglish>Bnlang File Paths</I18nEnglish>
# <I18nBangla>Bnlang ফাইল পাথ</I18nBangla>


<I18nEnglish>
Working with **file paths** is an important part of file system operations in Bnlang.  
File paths can be **absolute** (starting from the root of the file system) or **relative** (starting from the current working directory).  
Bnlang provides built-in modules to handle file paths in a cross-platform way, so your programs work on Windows, Linux, and macOS.
</I18nEnglish>

<I18nBangla>
Bnlang এ **ফাইল পাথ** নিয়ে কাজ করা ফাইল সিস্টেম অপারেশনের একটি গুরুত্বপূর্ণ অংশ।  
ফাইল পাথ হতে পারে **অ্যাবসোলিউট** (ফাইল সিস্টেমের রুট থেকে শুরু) অথবা **রিলেটিভ** (বর্তমান কাজের ডিরেক্টরি থেকে শুরু)।  
Bnlang ক্রস-প্ল্যাটফর্ম উপায়ে ফাইল পাথ হ্যান্ডল করার জন্য বিল্ট-ইন মডিউল সরবরাহ করে, যাতে আপনার প্রোগ্রাম Windows, Linux এবং macOS এ কাজ করে।
</I18nBangla>

## Types of Paths

<I18nEnglish>
- **Absolute Path** → Specifies the full path from the root of the file system.  
- **Relative Path** → Specifies the path relative to the current working directory.  
- **Cross-Platform Paths** → Use helpers to handle different separators (e.g., `/` in Linux, `\` in Windows).
</I18nEnglish>

<I18nBangla>
- **অ্যাবসোলিউট পাথ** → ফাইল সিস্টেমের রুট থেকে পুরো পাথ উল্লেখ করে।  
- **রিলেটিভ পাথ** → বর্তমান কাজের ডিরেক্টরি থেকে পাথ উল্লেখ করে।  
- **ক্রস-প্ল্যাটফর্ম পাথ** → আলাদা সিস্টেমের সেপারেটর (যেমন Linux এ `/`, Windows এ `\`) হ্যান্ডল করার জন্য হেল্পার ব্যবহার করুন।
</I18nBangla>

## Best Practices

<I18nEnglish>
- Always use the `path` module for portability.  
- Avoid hardcoding separators (`/` or `\`) in your code.  
- Use `path.resolve()` for absolute paths and `path.join()` for building relative paths.  
- Test file operations on multiple platforms if your project targets cross-platform environments.
</I18nEnglish>

<I18nBangla>
- সর্বদা `path` মডিউল ব্যবহার করুন যাতে পোর্টেবিলিটি বজায় থাকে।  
- কোডে সেপারেটর (`/` বা `\`) হার্ডকোড করা এড়িয়ে চলুন।  
- অ্যাবসোলিউট পাথের জন্য `path.resolve()` এবং রিলেটিভ পাথ তৈরির জন্য `path.join()` ব্যবহার করুন।  
- যদি আপনার প্রজেক্ট ক্রস-প্ল্যাটফর্ম হয় তবে একাধিক প্ল্যাটফর্মে ফাইল অপারেশন টেস্ট করুন।
</I18nBangla>
